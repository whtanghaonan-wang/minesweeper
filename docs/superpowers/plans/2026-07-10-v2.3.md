# 扫雷 v2.3 总实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变扫雷规则、50 关数表和 SaveData schema v3 的前提下，交付强 Liquid Glass、果冻动效、完整无障碍/移动端修复、可靠存档、PWA 安全更新和可复现发布链，并正式发布 v2.3.0。

**Architecture:** 规格横跨四个可独立验收的子系统，因此按依赖顺序拆成四份实施计划。核心可靠性先固定存储与无尽模式契约；无障碍/移动视口随后重建棋盘 DOM 与焦点模型；Liquid Glass 在稳定 DOM 上叠加单层光学表面和事件控制器；PWA/安全/发布最后接入更新协调器、版本权威源、Tauri/Actions 闸门。

**Tech Stack:** TypeScript 6、Vite 8、Vitest 4 + jsdom、Playwright（Chromium/WebKit）+ axe、vite-plugin-pwa/Workbox、Tauri 2、GitHub Actions、PowerShell。

## 3. 全局执行规则

- 每个行为变更严格执行 RED → GREEN → REFACTOR：先写失败测试并确认失败，再写最小实现。
- 源文件只用 `apply_patch` 编辑；格式化、Cargo lockfile 和 npm lockfile 的机械更新可由对应工具产生。
- 每个任务只提交列出的相关文件；不要带入 `.superpowers/`、安装包、`dist/`、`target/` 或用户的无关变更。
- 每个任务结束至少运行该任务的定向 Vitest；每个子计划结束运行 `npm test` 与 `npm run build:web`。
- 影响桌面配置后运行 `npm run build:desktop`；正式候选还必须运行 Tauri NSIS 构建。
- 不改生成器、求解器、关卡表和既定音效参数；发现相关回归立即停下定位。
- 没有证书时允许产生未签名候选，但必须如实记录 `NotSigned`；不得把“签名入口已配置”描述为“已签名”。
- 未完成 iOS 27 / Windows 候选验收前，不创建正式标签。
