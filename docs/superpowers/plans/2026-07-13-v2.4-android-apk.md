# v2.4 Signed Android APK Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and validate a production-signed Android 7+ universal APK from the existing Tauri 2 project, distribute a same-SHA candidate through GitHub Actions, and prepare a verified unified v2.4.0 release handoff.

**Architecture:** Tauri's generated Android Gradle target remains under `src-tauri/gen/android`; a committed platform config supplies API 24 and deterministic version code. A Node artifact script performs release build packaging and invariant checks, while an Android candidate/tag workflow reconstructs a temporary signing keystore from GitHub environment secrets and uploads only verified artifacts. GitHub Release creation remains a post-device-validation operation using tag artifacts from Windows and Android workflows.

**Tech Stack:** Tauri CLI 2.11.4, Rust 1.96.1, Android Studio/JBR 17, Android SDK Platform 36, Build-Tools 36.0.0, NDK 27.2.12479018, Gradle generated by Tauri, Node.js 22, GitHub Actions, `apksigner`, `aapt`, `keytool`.

```js
const androidVersionCode = (semver) => {
  const match = /^(\d+)\.(\d+)\.(\d+)$/.exec(semver);
  if (!match) throw new Error(`Android versionCode requires x.y.z version, got ${semver}`);
  const [major, minor, patch] = match.slice(1).map(Number);
  const code = major * 1_000_000 + minor * 1_000 + patch;
  if (!Number.isSafeInteger(code) || code < 1 || code > 2_100_000_000) {
    throw new Error(`Android versionCode out of range: ${code}`);
  }
  return code;
};
```
