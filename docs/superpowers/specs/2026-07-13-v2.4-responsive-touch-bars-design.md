# Minesweeper v2.4 响应式胶囊触控条设计

日期：2026-07-13
状态：用户已确认方案 A

## 目标

解决 v2.3 首页底部菜单在桌面端呈现为高大圆角方卡、手机游戏顶部玻璃栏过宽且光学效果过重的问题。v2.4 将首页菜单重构为随设备形态变化的单层 Liquid Glass 大 R 角触控条，并将手机游戏顶栏收窄、减轻视觉重量，同时保持现有功能、无障碍与存档兼容。

## 设计方向

整体延续藤蔓、纸色背景和绿色主操作的有机自然风格。玻璃只负责承载和分组，不作为厚重卡片；内部按钮使用现有 jelly 触感，不叠加第二层 backdrop blur。

## 首页触控条

### 桌面端

- `home-panel` 保持唯一 Liquid Glass 表面，但从竖向卡片变为低矮横向胶囊，最大宽度约 54rem，高度由内容决定且目标控制在 7–8rem。
- 触控条使用四个逻辑区域：进度统计、绿色主操作、选关/无尽次操作、声音/透明度工具。
- 统计区包含通关数、最近通关和细进度线；它们不再占据整条宽度。
- “继续/开始游戏”是视觉中心，保持高对比实色绿色胶囊。
- 版本号移到触控条外部下方，避免增加玻璃表面高度。
- 在超宽屏上保持固定最大宽度，不随窗口拉伸成横贯屏幕的大条。

### 手机端

- 沿用同一 DOM 和信息顺序，使用内容驱动断点折成紧凑双层大胶囊。
- 顶部一行容纳统计和声音/透明度工具；主操作独占中间一行；选关与无尽在底部并排。
- 宽度以视口为基准并保留安全区，所有按钮至少 44×44px。
- 在 320px 宽和 200% 根字号下允许内容换行，但不允许页面横向溢出。

## 手机游戏顶栏

- 保留“返回+标题”第一行和“炸弹/时间/声音/重开”四等分第二行，不改变语义与操作顺序。
- 390px 视口目标宽度约 308px；小于 360px 时允许回宽至约 92vw，避免数字和 44px 触控区冲突。
- 使用专用轻量玻璃修饰类降低 blur、saturation、棱镜层、高光和投影强度；桌面顶栏保持单行紧凑布局。
- 手机顶栏居中，不使用 `margin-left:auto` 推挤统计区；四个第二行槽位保持等宽。

## 兼容性与无障碍

- 不改变棋盘、关卡、存档结构、底部操作栏或 PWA 更新状态机。
- 保留键盘焦点、ARIA、forced-colors、prefers-reduced-motion 和 prefers-reduced-transparency 回退。
- 玻璃表面仍禁止嵌套，棋盘与格子不增加 filter/backdrop-filter。
- v2.3 存档直接复用，不新增迁移。

## 版本与发布

- 将 npm、Cargo、Tauri 和锁文件版本统一升级到 `2.4.0`。
- 先推送候选 SHA 到 `main` 并部署 Pages，等待用户在实体 iPhone 和 Windows 桌面复验。
- 复验通过后创建同 SHA 的 `v2.4.0` 标签和 GitHub Release。
- Windows 产物继续明确标记代码签名状态；未配置证书时为 `NotSigned`。

## 验收标准

- 桌面首页不再出现高大圆角方卡，控制区呈明显低矮横向胶囊。
- 手机首页仍可完整操作，主操作突出，320px 与 200% 字号无横向溢出。
- iPhone 390px 游戏顶栏宽度约 308px，视觉效果比 v2.3 更轻，四个状态/操作槽位对称且均不小于 44px。
- Chromium 与 WebKit E2E、axe、PWA 更新测试、Web/Desktop 构建、版本一致性检查全部通过。
