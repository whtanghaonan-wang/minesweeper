# Minesweeper v2.4 首页液化玻璃选中反馈设计

日期：2026-07-13
状态：视觉原型 v8 已验收，待书面规格复核

## 背景与目标

v2.4 首页已经改为一整块大 R 角 Liquid Glass 触控条，但内部主按钮、次按钮和工具按钮仍各自带有背景或边框，形成多层胶囊并导致窄屏溢出。现有按压反馈也只是按钮自身缩放，无法表现参考视频中“同一团液化玻璃在面板内流动、变形并吸附到不同菜单”的效果。

本次改动只重做首页触控条内部的选中反馈：保留唯一外层玻璃面板，在其内部增加一团持久存在、尺寸自适应、可点击移动也可鼠标/手指拖动的液化玻璃选中体。它与外层面板属于同一材质，只通过略深的折射和色彩浓度突出当前位置。

## 范围

参与交互的目标共有五个：

- “开始游戏/继续”主操作；
- “选关”；
- “无尽”（仅解锁时可用）；
- 声音开关；
- 透明度开关。

统计信息和进度条保持只读，不成为吸附目标。棋盘、游戏页顶部/底部操作栏、关卡逻辑、存档结构、PWA 更新和版本号不在本次改动范围内。

## 视觉结构

### 单层玻璃

- `.home-panel` 继续是首页唯一带 `data-liquid-glass` 的光学表面。
- 五个按钮在空闲状态全部取消独立背景、边框、阴影和 backdrop blur，只保留文字、图标及不小于 44×44px 的透明触控区。首页这五个目标不再使用通用 `[data-jelly]` 缩放，避免文字和图标随按钮单独缩小；按压、选中和拖拽反馈全部由新的首页控制器统一完成。
- 面板内部增加一个 `aria-hidden="true"` 的绝对定位液化选中体。它不是第二个 `data-liquid-glass` 表面，不形成嵌套玻璃，也不改变按钮的 DOM 语义或 Tab 顺序。
- 选中体位于文字和图标之后、面板光学层之上；统计和按钮内容始终清晰可读。

### 材质与颜色

- 选中体沿用外层面板的白色半透明玻璃基色、模糊度和边缘高光，只将明度略压低、饱和度略提高，使其像同一块玻璃内部聚集出的液滴。
- 折射色限定为低浓度绿色、青色和浅紫色，不使用粉色，也不使用独立实色块。
- 默认由选中体包裹“开始游戏/继续”，被选中的按钮文字和图标使用系统蓝色；其他按钮保持现有深色文字。选中状态不能代替 `:focus-visible` 焦点环。
- 指针按下和拖动时继续更新外层面板的 `--glass-x`、`--glass-y`、`--glass-dx`、`--glass-dy`，让周围大胶囊的折射高光与液滴实时联动，而不是只有内部选中体在运动。

### 自适应几何

- 每次定位都读取目标按钮的实际边界，并在四周各增加 5 CSS px 光学余量；胶囊宽高不写死。
- “开始游戏”形成较宽的长胶囊，声音等工具按钮形成紧凑圆胶囊，选中体在移动过程中连续变形到目标尺寸。
- 选中体始终限制在 `.home-panel` 内，不覆盖面板外版本号，也不造成页面横向滚动。
- 窗口缩放、横竖屏切换、字号变化或布局断点切换时，取消旧动画并按当前目标重新测量、落位。

## 点击与键盘行为

- 点击未选中的可用按钮时，选中体从当前位置压缩、拉伸并滑向目标，目标文字和图标同步变蓝，旧目标恢复深色。
- 点击声音或透明度开关时，原功能立即执行，选中体留在该按钮并完成 Q 弹回稳。
- 点击会离开首页的“开始游戏/继续”“选关”和已解锁“无尽”时，仍然只需一次操作；若目标原本未选中，视觉反馈先启动并在 220ms 后执行原导航，若目标已经选中则立即执行，避免重复等待。
- Enter 和 Space 触发与点击相同的选中和原功能，但不进入拖拽状态；现有键盘焦点顺序、ARIA 标签和开关的 `aria-pressed` 保持不变。
- 锁定的“无尽”保持禁用状态，不变蓝、不吸附、不能通过拖拽触发。

## 拖拽与吸附行为

### 开始和移动

- 只能从当前选中按钮或选中体露出的边缘开始拖拽；未选中按钮仍保留普通点击行为。
- 统一使用 Pointer Events，因此鼠标和单指触摸走同一状态机。非主指针和鼠标非左键被忽略。
- 拖动中按速度改变横纵缩放：沿运动方向轻微拉长、垂直方向轻微压缩，不旋转，不生成脱离面板的实体飞行胶囊。
- `pointermove` 以 `requestAnimationFrame` 合并更新，位置、速度形变和候选目标在同一帧计算。

### 磁性目标和释放

- 指针靠近可用按钮时进入磁性范围，选中体逐步靠向该按钮并连续适配目标尺寸；候选文字可以提前变蓝。
- 在不同的候选目标上释放时，选中体以轻微越界、回弹、稳定三段完成落位，并按与点击相同的规则触发该按钮原有功能：工具开关立即执行，未选中的导航目标在 220ms 后执行。
- 拖回当前目标后释放只回稳、不重复触发原功能，避免用户调整拖拽时误进游戏或误切开关。
- 在空白区域或锁定按钮附近释放时，不触发功能，选中体回到拖拽前的当前目标。
- 完成一次落位后无需重新点击，立即可以从新位置再次拖动。

## 动效

- 点击移动采用 580ms 的液体滑动：旧位置先收缩，主体沿面板平滑伸展到目标，最后用一次小幅 squash/stretch 回稳。
- 拖拽释放采用 420ms 的 Q 弹落位；实时拖动阶段直接跟手，不添加滞后的预设轨迹。
- 全程不旋转选中体，不使用粉色闪光，不让选中体与外层面板看起来是两种材质。
- `prefers-reduced-motion: reduce` 时取消路径拉伸和回弹，改为不超过 100ms 的位置/透明度切换；交互结果和原功能不变。

## 组件与生命周期

新增一个仅服务首页的液化选中控制器 `home-liquid-selection.ts`：

- 输入为首页面板、默认按钮，以及包含按钮、是否为导航操作和原业务回调的五个目标描述；
- 负责选中状态、尺寸测量、点击移动、拖拽状态、磁性命中、动画和清理；
- 只在一次活动拖拽期间向 `window` 注册 move/up/cancel/blur 兜底监听，拖拽结束立即移除；
- 暴露 `destroy()`，用于页面离开、首页重建和测试清理；
- 不修改通用 `liquid-glass.ts` 的按压/高光职责，也不影响其他 `data-jelly` 元素。

`home.ts` 负责创建选中体、构造五个目标及其原业务回调，并返回包含 `destroy()` 的首页视图控制器；按钮不再各自注册一套会绕过动画的 click 监听。`main.ts` 保存当前首页视图控制器，在进入首页、选关、游戏或无尽模式前统一销毁旧实例。业务回调仍由现有依赖注入提供，液化选中控制器不读取关卡或存档。

## 中断恢复与错误处理

拖拽状态以 `pointerId` 标识，但不能假设每次都能收到面板内的 `pointerup`。以下入口必须统一清理当前会话、候选样式、动画帧和捕获状态：

- `pointerup`；
- `pointercancel`；
- `lostpointercapture`；
- `window.blur`；
- 页面进入 hidden；
- 控制器销毁或布局重建。

结束事件在活动拖拽期间由 `window` 捕获，避免手指在面板外抬起时丢失清理。`setPointerCapture`、`hasPointerCapture` 和 `releasePointerCapture` 均使用异常保护；即使浏览器不支持或中途丢失捕获，全局结束监听仍能完成清理。

新的主指针按下时如果发现遗留会话，先无条件回收旧 `pointerId` 再开始新会话。这一防线确保一次系统手势中断不会让后续触摸永久退化成只能点击。

Web Animations API 或有效布局尺寸不可用时直接落到目标最终样式，不阻断按钮原功能。目标若在计划触发业务回调之前被外部移除，则取消动画和尚未触发的回调；由该回调正常导航并移除首页不属于错误。

## 无障碍与降级

- 选中体纯装饰、对辅助技术隐藏；按钮继续提供真实语义、禁用状态、名称和焦点。
- `prefers-reduced-transparency: reduce` 或应用“实色”模式下，选中体改为轻微加深的浅绿色/灰绿色实色，不再使用 backdrop blur。
- `forced-colors: active` 下关闭装饰性折射，用系统 `Highlight`/`HighlightText` 或清晰轮廓表示当前目标，同时保留独立焦点环。
- 200% 根字号、320px 宽度和中英文长度变化时，选中体按实际目标重新测量，不裁切文字、不减小触控区。

## 测试策略

### 单元测试

- 首页 DOM 仍只有一个 `data-liquid-glass`，五个按钮没有独立玻璃，选中体默认定位到主操作。
- 点击各可用按钮更新选中状态且原回调只触发一次；锁定“无尽”不可选中。
- 拖拽到可用目标、空白区域和锁定目标分别验证落位、回退和不触发。
- 使用不同目标边界验证选中体宽高随按钮变化，而不是固定尺寸。
- 先以旧 `pointerId` 开始，再派发 `lostpointercapture`，随后用新 `pointerId` 拖拽，验证第二次仍能完成切换。
- 覆盖面板外释放、`pointercancel`、窗口失焦、页面隐藏、重复快速拖拽、动画取消和 `destroy()`。
- 覆盖 reduced-motion、reduced-transparency 和 forced-colors 的静态规则。

### 端到端测试

- Chromium 和 WebKit 在 390×844、320×844、桌面宽屏及 200% 根字号下验证所有按钮和选中体不溢出面板。
- 在手机尺寸连续交替拖拽至少 20 次，包含从小胶囊边缘再次抓取，验证不会退化为只能点击。
- 强制在拖动途中释放 pointer capture 并在面板外结束，再以新 pointerId 拖拽，验证中断自恢复。
- 验证点击导航按钮仍只需一次操作，声音/透明度状态和存储行为不回归。
- 完整运行 Vitest、Chromium/WebKit E2E、axe、PWA 更新测试、Web/Desktop 构建、Cargo 检查、版本一致性检查和 `git diff --check`。

## 验收标准

- 首页内部不再出现五个独立白色或绿色实体胶囊；始终只有一团与面板同材质、略深的液化玻璃选中体。
- 选中体默认包裹“开始游戏/继续”，自动适配每个目标尺寸，选中文字和图标为系统蓝色。
- 点击和拖拽都能让选中体沿面板丝滑移动，并以无旋转的果冻 Q 弹效果落位。
- 鼠标和手指都可连续拖拽；即使某次在面板外抬手、丢失捕获或切换前后台，后续拖拽仍可立即使用。
- 锁定“无尽”不可吸附，空白释放回原位，原有按钮功能、存档、无障碍和响应式约束保持正确。
