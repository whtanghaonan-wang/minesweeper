# 扫雷 v2 设计：关卡扩充 · 雷区缩放平移 · 藤蔓选关 · 响应式 · 插旗修复

日期：2026-07-03 ｜ 状态：已获用户确认 ｜ 前置：v1 设计 `2026-07-02-minesweeper-design.md`

## 0. 背景与目标

v1（10 关 3 档）上线后用户反馈：关卡少、偏简单；PC 端感觉没有插旗；PC 全屏时雷区不放大、留大片空白。v2 四项更新：

1. 关卡扩到 **20 关 5 档**，顶档难度质变；配套**雷区缩放 + 平移**，保证大盘在固定屏幕上可丝滑操作，并有防误触判定。
2. 选关首页改为**树枝状（蜿蜒藤蔓）布局**，深度与难度成正比，档位交界即拐点（用户已在视觉稿中选定 A·蜿蜒藤蔓）。
3. PC 端 UI **随窗口尺寸等比缩放**，全屏不留大面积空白。
4. PC 端**右键插旗**修复与操作提示。

技术底座不变：TypeScript + Vite 无框架，PWA（GitHub Pages）+ Tauri 2 exe。

## 1. 关卡体系（20 关 · 5 档）

### 1.1 生成器性能实测（设计依据）

对候选规格各生成 10 盘（随机首点，现有同步生成器）：

| 规格 | 密度 | 中位耗时 | 最差 |
|---|---|---|---|
| 16×30@99（经典专家） | 20.6% | 2ms | 9ms |
| 20×34@136 | 20.0% | 9ms | 24ms |
| 20×34@150 | 22.1% | 43ms | 137ms |
| 22×38@190 | 22.7% | 134ms | 331ms |
| 22×38@205 | 24.5% | 910ms | 5.7s |
| 24×42@252 | 25.0% | 3.6s | 15.1s |

结论：**密度 ≤22% 毫秒级，≥24% 秒级**。顶档定 22.1%，同步生成保留（无需 Worker）；`generate` 的 `onSlow` 钩子保留作保险。

## 2. 视口模块：缩放 · 平移 · 防误触

新增 `src/ui/viewport.ts`：纯逻辑手势/变换状态机（可单测），`game.ts` 只做 DOM 接线。棋盘 DOM 结构不变，外包视口容器，对 `.board` 施加 `transform: translate(tx,ty) scale(s)`（`transform-origin: 0 0`）。

### 2.1 变换模型

- **适配缩放 `fitScale`**：进关/窗口尺寸变化时计算，使整盘恰好铺满棋盘可用区（含内边距）。初始 `s = fitScale`。
- 缩放范围：`s ∈ [fitScale, max]`，`max` = 单格屏显约 64px 对应的比例。
- **定点缩放**：`zoomAt(px, py, factor)` —— 以屏幕点 (px,py) 为不动点更新 s、tx、ty。
- **平移钳制**：盘小于视口的轴向居中；盘大于视口的轴向 `tx ∈ [viewW - boardW·s, 0]`（y 同理），雷区永不甩出屏幕。
- 基础格尺寸固定 CSS 值（如 40px），一切大小变化都由 transform 完成，避免反复重排。

### 2.2 输入判定状态机（防误触核心）

指针事件统一处理（Pointer Events），挖/旗一律在**抬起时**结算：

- 点按语义与 v1 一致（不分模式）：未开格 = 挖/旗（按输入方式），已开数字格 = 尝试 chord，旗格点按无操作。
- **鼠标**：左键按下→若累计位移 < 4px 且未被中断，抬起时执行点按动作；≥ 4px 判定为**拖动平移**，本次手势永不触发格子动作。滚轮 = 以鼠标位置定点缩放。**右键 = 插旗**（按下即生效，不参与平移判定）；棋盘上下文菜单继续屏蔽。
- **触摸**：单指按下启动长按计时（350ms 长按 = 反模式动作）；位移 ≥ 10px 判定为**拖动平移**，取消长按计时与点按；抬起时若未拖动未长按 → 执行当前模式动作。**双指落下即进入捏合态**：立即取消一切点按/长按意图，两指中点定点缩放 + 双指平移；**捏合态结束后进入冷却：残留手指的抬起不触发任何格子动作**，直到全部手指离开。
- 视口容器 `touch-action: none`，手势全部自管；`prefers-reduced-motion` 尊重不变。

### 2.3 与现有行为的整合

- 宽屏（w>h）行列转置保留，转置发生在渲染层，视口变换作用于视觉网格，互不干扰。
- 揭格连锁动画、win/lose 全盘展示等渲染逻辑不变。
- 模式切换（⛏/🚩）仍只在触摸端显示；鼠标端左挖右旗固定。

## 3. 藤蔓选关页（用户选定 A·蜿蜒藤蔓）

`menu.ts` 重写为竖向滚动藤蔓地图：

- 结构：滚动容器内一张 SVG（藤蔓路径 + 档位色带）+ 绝对定位的关卡节点按钮层（保留 DOM 按钮：焦点、aria、点击不变）。
- 布局生成：`vine.ts` 纯函数按关卡数产出蜿蜒锚点（正弦式左右摆动、自下而上），节点等距分布；档位交界处色带切换形成拐点。根部 🌱、顶端 👑。
- 节点状态：已通关 = 档位色实心圆 + 白字关号 + 下方小字最好成绩；当前关 = 白底 + 档位色描边 + 光圈呼吸动画；未解锁 = 虚线灰圆 + 🔒不可点。
- 打开选关页自动滚动到当前关附近（当前关居于视口下 1/3 处）。
- 顶部标题区保留；副标题改「无猜 · 二十关 · 五档」（PWA manifest 描述同步改）。
- 逐关解锁规则不变。

## 4. 全窗口等比响应式（PC 为主，手机同样受益）

- `:root` 字号 `clamp(14px, 1.1vmin + 9px, 26px)`（初值，实施时按观感微调），UI 尺寸令牌（内边距、圆角、按钮、标题等）全面 px→rem：窗口越大整套 UI 同比例放大。
- 雷区依靠视口模块的 fitScale 始终铺满棋盘区（解决全屏留白的主因）。
- 选关藤蔓地图宽度上限同样以 rem 定义，随窗口缩放。
- Tauri 窗口：默认 480×760 → **1100×800**，最小 640×560；标题不变。

## 5. 右键插旗修复

v1 排查结论：`game.ts` 已有右键插旗（pointerdown button===2），但**盘面生成前（首挖前）右键被 `board === null` 分支静默忽略**，令用户认为无此功能。修复：

- **开局前可插旗**：预旗集合（Set）记录首挖前的旗位并正常渲染；首挖生成盘面后把预旗原样落到盘上（旗不影响无猜生成，仅是标记）；预旗格与普通旗一致：不可被挖、可再次右键取消。剩余雷数显示同步计入预旗。
- 底部（仅 fine pointer 显示）加操作提示行：`左键挖开 · 右键插旗 · 滚轮缩放 · 拖动平移`。
- 验收含桌面 exe 实测右键（防 WebView 差异）。

## 6. 存档迁移（v1 → v2）

`SaveData` 升 `version: 2`，`MAX_LEVEL` 改由关卡表长度导出（20）：

- `unlockedLevel` 原样继承（1..10 映射不变，上限放宽到 20）。
- `bestTimes` 仅保留**盘面规格（W×H×雷数）与 v1 完全一致**的关：即第 1、2 关；其余清空（盘面已变，旧成绩不可比）。迁移函数内嵌 v1 规格表做比对。
- 读到 version 2 走正常校验；读到 version 1 走迁移；其它一律回默认。迁移后立即持久化。

## 7. 测试策略

新增（全部 Vitest，纯逻辑不依赖浏览器的进 node 环境，DOM 冒烟进 jsdom）：

1. `viewport` 数学：fitScale、钳制、zoomAt 不动点性质、缩放边界。
2. 手势状态机：位移阈值内点击/超阈值拖动不触发、双指进入捏合即取消点按、捏合冷却期抬指不触发、长按与拖动互斥。
3. 关卡表校验：20 关、每档 4 关、密度档内单调递增、档间跳变 ≥ 既定阈值、限时单调不减。
4. 无猜性质抽样：新规格中抽代表关（含第 20 关）各生成若干盘验证可解且毫秒级预算。
5. 存档迁移：v1 存档各字段继承/清空规则、损坏数据回退。
6. 藤蔓菜单 jsdom 冒烟：节点数、锁定态、点击进关、自动滚动调用。
7. 预旗：开局前右键→旗渲染与雷数联动→首挖后旗保留。

原 47 个测试保持通过（levels 相关断言随新表更新）。
