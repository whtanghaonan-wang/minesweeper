## 1. 点击可靠性修复

### 1.1 根因(源码定位,确定性逻辑)

- `viewport.ts` `MOUSE_SLOP_PX = 4`:按下→抬起间移动 ≥4px 即判定拖动平移,抬起不挖格。快速用力点击常微移 4px 以上,点击被吃掉。
- 格距 3px + 雷区内边距 10px 是命中死区:`cellIndex(e.target)` 靠 DOM 目标找格,点在缝上返回 null,无任何响应。

### 1.2 方案

- **几何吸附命中**:新增纯函数(放 `viewport.ts`):

  ```
  hitCell(px, py, view: ViewState, cols, rows): number | null
  ```

  视口坐标反算棋盘坐标 `bx = (px - view.tx) / view.scale - BOARD_PAD`(y 同理),按 43px 栅距(40 格 + 3 缝)定位格子;点落在格内直接命中;落在缝隙/边距时,若距最近格边缘 ≤2px(棋盘坐标系)则吸附到该格,否则返回 null(留给拖动平移)。3px 缝隙内任意点距邻格 ≤1.5px,全部被吸附覆盖,死区清零。
- `game.ts` pointerdown 改用 `hitCell` 替代 `closest('.cell')` 求 `downCellVi`;"按下点决定挖哪格"的语义不变;右键插旗与长按同样受益。
- `MOUSE_SLOP_PX` 4 → **8**;`TOUCH_SLOP_PX` 10 不变(长按判定依赖)。

### 1.3 验收

- 点在任意缝隙/贴边处必命中最近格;按下后移动 <8px 抬起必挖格;拖动平移手感不明显变化(≥8px 才起拖)。

## 3. 音效(A·木质柔和)

### 3.1 模块

新建 `src/core/audio.ts`:Web Audio 实时合成,零音频素材文件。接口:

```
initOnGesture()      // 首次用户手势时惰性创建 AudioContext(自动播放策略)
playBlank() playNumber() playBoom() playWin() playLose()
setMuted(b) / isMuted()
```

### 3.2 合成参数(与用户试听页一致,听到什么上线什么)

包络:linear attack 2ms → exponential decay;`tone(freq, type, dur, peak, when, bendTo)`、`noise(dur, peak, when, lowpass, lpEnd)`:

- **空白格**:sine 520Hz/90ms/0.22 + sine 1040Hz/50ms/0.06 + 低通 1200Hz 噪声 30ms/0.06;
- **数字格**:sine 760Hz/80ms/0.2 + sine 1520Hz/40ms/0.05;
- **触雷爆炸**:噪声 500ms/0.5(低通 900→120Hz)+ sine 90→45Hz/400ms/0.5;
- **通关**:triangle 琶音 523/659/784/1047Hz,各 160ms/0.18,间隔 90ms;
- **失败**:triangle 下行 392/311/262Hz,各 220ms/0.2,间隔 160ms。

### 3.3 触发规则

- 一次挖掘只播一次:挖开格邻雷数为 0(连锁展开)→ 空白格音;≥1 → 数字格音;
- 踩雷:触雷瞬间播爆炸音,结算弹窗出现时(FINISH_PAUSE 后)播失败音;
- 超时失败:仅结算时播失败音;
- 通关:`win()` 时播通关音;
- 插旗/预旗/取消旗:无音效(保持克制,用户未要求)。

### 3.4 静音开关与持久化

- 首页与游戏内顶栏各一个 🔊/🔇 按钮;
- `SaveData` 增可选字段 `soundOn?: boolean`,缺省视为 `true`;存档 `version` 保持 2,`sanitize` 容缺省,不需要迁移;
- Tauri(WebView2)与浏览器同一 Web Audio 路径,无平台分支。

## 4. 游戏首界面(B·藤蔓延伸式)

### 4.1 页面流

```
启动(PWA/桌面)→ 首页 home
home:[继续·第N关] → game(当前进度关)    [🌿选关] → menu(藤蔓)
menu:点关卡 → game;返回 → home
game:退出 → menu;结算弹窗(下一关/重开/回选关)行为不变
```

### 4.2 布局与内容(新 `src/ui/home.ts`)

- 上半屏:装饰藤蔓 SVG 自顶垂落,`stroke-dashoffset` 生长动画约 2s(`prefers-reduced-motion` 时直接静态显示);大标题"扫雷"(字距拉开),副题"沿着藤蔓,一路向上";
- 下半屏白色大圆角面板:
  - 状态行:🌱 已通关 **x**/50 · ⏱ 最快 **m:ss** · 🔊/🔇 静音钮;
  - 进度条(x/50,easy 绿 #8fae8b);
  - 主按钮:未开始→"开始游戏 · 第 1 关";进行中→"**继续 · 第 N 关**"(N=当前解锁最新关);全通→"再战 · 第 50 关";
  - 副按钮:"🌿 选关";
  - 底部小字版本号 v2.1.0。
- "最快成绩"定义:**最高的已通关且有成绩的关卡**的最好用时;一个成绩都没有时显示"—";
- 皮壳沿用 rem 响应式体系,窄屏自然单列,无需特殊分支。
